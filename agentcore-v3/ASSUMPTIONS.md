# Допущения при подготовке к ЮKassa

_Создано автоматически при аудите и доработке проекта AgentCore._

## Допущения о бизнесе

1. **Тип бизнеса**: SaaS-подписка (AI-агенты как услуга). Нет физических товаров, нет доставки, нет складского учёта.

2. **Страна регистрации**: Казахстан (г. Алматы). Реквизиты и legal-тексты ориентированы на казахстанскую юрисдикцию (ТОО, БИН).

3. **Валюта**: Доллар США ($). Реквизиты в тенге (₸) не указаны, нужна конвертация или отдельный прайс.

4. **Платёжный сценарий**: Одностадийный (one-stage payment) — оплата → мгновенное списание. Two-stage (hold/capture) не требуется для SaaS-подписки.

5. **Чеки (54-ФЗ)**: Услуга оказывается через «Чеки от ЮKassa». Внешняя онлайн-касса не подключается.

6. **Способы оплаты**: Банковские карты (Visa/MC/МИР) + YooMoney. СБП, SberPay, T-Pay — опционально, не реализованы в MVP.

7. **Реальные товары/услуги**: На сайте представлены AI-агенты как цифровая услуга. Карточки сценариев (цветочный, салон, etc.) — демонстрационные примеры, НЕ физические товары.

## Допущения о безопасности

8. **SSL/HTTPS**: Сервер НЕ имеет SSL-сертификата. Для модерации ЮKassa необходимо настроить nginx + Let's Encrypt.

9. **Секреты в env**: Файл `apps/api/.env` содержит реальные ключи. Файл добавлен в `.gitignore`, но уже мог быть в истории git. Рекомендуется смена паролей.

10. **API-ключ Suggy**: `sk-suggy-...` используется для маршрутизации AI-моделей. Это сторонний сервис, не связанный с платежами.

## Допущения о данных

11. **Админ-панель**: Страницы Orders/Payments/Customers используют демо-данные. API эндпоинты для этих разделов частично реализованы (Stripe), частично в процессе (YooKassa).

12. **Модель данных**: В схеме Prisma есть `BillingTransaction`, но нет отдельных сущностей `Payment`, `Refund`, `Order`, `Customer`, `AuditLog`. В рамках MVP используется `BillingTransaction` + JSON в `Workspace.settings`.

13. **Роли пользователей**: RBAC не реализован. Все залогиненные пользователи имеют полный доступ.

## Допущения о контенте

14. **Legal-тексты**: Политика конфиденциальности, оферта, возврат — являются типовыми черновиками. Требуют проверки юристом и адаптации под реальную юрисдикцию.

15. **Реквизиты**: Все реквизиты на странице `/requisites` являются шаблонными. Поля БИН, ИИН, ОГРН, банковские реквизиты не заполнены.

16. **Отзывы**: Носят демонстрационный характер. Перед модерацией ЮKassa должны быть заменены на реальные.

## Допущения об интеграции

17. **YooKassa SDK**: Пакет `yookassa` установлен в `package.json`, но НЕ установлен на сервере. Сервер запускается без него (graceful degradation). Для полноценной работы нужно: `npm install yookassa` на сервере + заполнить `YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET_KEY` в `.env`.

18. **Stripe**: Работает полностью (checkout session + webhook). Используется как основной платёжный провайдер.

19. **Webhook URL**: Для YooKassa webhook должен быть доступен из интернета по HTTPS. Без SSL webhook не заработает.

20. **Return URL**: Для YooKassa redirect используется `CLIENT_URL/settings/billing`. Без HTTPS редирект может не работать в некоторых браузерах.

## Предположения о масштабе

21. **MVP-объём**: Текущая версия — рабочее демо/MVP. Не хватает: полноценного RBAC, аудит-логов, очередей фоновых задач, полноценного тестирования.

22. **Нагрузка**: Не проводилось нагрузочное тестирование. Для продакшена рекомендуется добавить кэширование (Redis), CDN, и горизонтальное масштабирование.
